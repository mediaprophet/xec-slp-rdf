This document specifies an ontology for representing contributions, financial and non-financial currencies, and transactions within the Knowledge Economics Project. The ontology provides a structured way to model various types of contributions, currencies, and economic interactions, enabling fair and transparent value exchanges in a decentralized knowledge economy.

This is an unofficial draft and is subject to change as the project evolves.

Introduction

The Knowledge Economics Project leverages blockchain technology, specifically eCash, to create a fair and transparent economy for knowledge work. This ontology serves as the semantic foundation for modeling contributions, financial and non-financial currencies, and transactions, facilitating decentralized tracking, verification, and compensation of knowledge work.

Scope

This ontology encompasses:

  • Detailed classifications of contributions, such as necessary, unsolicited, valued, and collaborative types.
  • Financial currencies, including micropayments, gifts, subscriptions, and various fees and taxes.
  • Non-financial currencies, such as time banks, loyalty points, social currencies, and biosphere- and sociosphere-related credits.
  • Transactions and rewards linking contributions to currencies.

It aims to support fair compensation and transparent contribution tracking, focusing on semantic modeling compatible with cryptographic and blockchain tools.

Terminology

Contribution
Knowledge work or resources provided by an agent, characterized by type, duration, and dependencies.
Currency
A medium of exchange, either financial (e.g., eCash) or non-financial (e.g., time credits).
Financial Currency
Monetary currencies, including fiat and cryptocurrencies.
Non-Financial Currency
Alternative value representations, such as reputation points or environmental credits.
Transaction
An exchange or transfer of currency between agents.
Reward
Compensation linked to a contribution, using financial or non-financial currency.
Agent
An individual or organization participating in the knowledge economy.

Ontology Structure

The ontology uses RDF and OWL to define classes and properties, ensuring interoperability with web standards and extensibility for future needs.

Classes

Core classes include:

  • Agent: Participants in the knowledge economy.
  • Project: Contexts for contributions.
  • Contribution: Work or resources contributed, with detailed characteristics.
  • Currency: Superclass for all currencies.
    • FinancialCurrency: Monetary currencies.
      • FiatCurrency
      • Cryptocurrency (e.g., eCash)
    • NonFinancialCurrency: Alternative currencies.
      • BiosphereCurrency: Environmental currencies (e.g., WaterCredit).
      • SociospheresCurrency: Social currencies (e.g., VolunteerCredit).
      • OtherNonFinancialCurrency: General alternatives (e.g., LoyaltyPoint).
  • Transaction: Currency exchanges.
  • Reward: Compensation for contributions.
  • Evaluation: Contribution assessments.

Contribution Classifications

Contributions are modeled with a single Contribution class, using properties to capture classifications:

  • Necessary Contributions: Essential tasks, marked by isEssential.
  • Unsolicited Contributions: Voluntary efforts, indicated by isRequested (false).
  • Valued Contributions: High-value work, assessed via Evaluation.
  • Collaborative Contributions: Multi-agent efforts, with multiple contributor links.
  • Time-limited Contributions: Short-term tasks, defined by startDate and endDate.
  • Ongoing Contributions: Long-term efforts, with extended or indefinite duration.
  • Discrete Contributions: Standalone tasks, with no dependencies.
  • Interdependent Contributions: Linked tasks, with dependencies.
  • Different Types of Work: Categorized by typeOfWork (e.g., design, research).
  • Contributions of Resources or Equipment: Tracked by resourcesUsed.
  • Contributions of Intellectual Property: Noted by intellectualProperty.Contributions on Defined Terms: Specified by terms.
  • Expertise-based Contributions: Defined by expertiseArea.
  • Role-based Contributions: Linked to role.
  • Standalone Contributions: Synonymous with discrete, no dependencies.

Currencies

Financial Currencies

Modeled under FinancialCurrency, with transaction types specified via Transaction properties:

  • Micropayments, Gifts, Donations, Subscriptions, Installment Payments, Salaries/Wages, Royalties, Licensing Fees, Commissions, Dividends, Rent, Loans, Grants, Scholarships, Bursaries, Insurance Payments, Tips
  • Taxes: Sales Tax, VAT, Property Tax, etc.
  • Fees: Usage Fees, Transaction Fees, Service Charges, etc.
  • Other: Security Deposits, Lease Payments, etc.

Non-Financial Currencies

Organized under NonFinancialCurrency with subclasses:

  • General: Time Banks, Loyalty Points, Social Currency, Alternative Currencies, Reputation Points, Coupons, Gift Cards, Air Miles, Points, Credits, Tickets, Vouchers, Tokens, Community Currencies, Virtual Currencies
  • Biosphere: Water Credits, Biodiversity Credits, Carbon Offset Credits, Ecological Credits, Nature Credits, Eco-points, Green Dollars, Renewable Energy Credits, Waste Credits, Green Bonds, Eco-tokens, etc.
  • Sociosphere: Social Impact Bonds, Volunteer Credits, Time Banks, Community Service Credits, Social Points, Impact Tokens, etc.

Properties

  • Contribution:
    • contributor: Links to Agent
    • project: Links to Project
    • typeOfWork: e.g., "design"
    • resourcesUsed: Contributed resources
    • intellectualProperty: IP details
    • terms: Licensing terms
    • expertiseArea: Expertise domain
    • role: Contributor role
    • dependencies: Linked contributions
    • startDate, endDate: Duration
    • isEssential: Boolean
    • isRequested: Boolean
  • Currency:
    • name, description, issuer
  • Transaction:
    • sender, receiver: Agent
    • currency: Currency
    • amount: Decimal
    • date: DateTime
    • purpose: e.g., "micropayment"
  • Reward:
    • forContribution: Contribution
    • currency, amount, date
  • Evaluation:
    • evaluates: Contribution
    • evaluator: Agent
    • score: Decimal
    • comments: String

Examples

A Turtle example of a collaborative contribution rewarded with eCash:

@prefix ke:  .
@prefix xsd:  .

ke:contrib1 a ke:Contribution ;
  ke:contributor ke:alice, ke:bob ;
  ke:project ke:proj1 ;
  ke:typeOfWork "research" ;
  ke:startDate "2025-01-01"^^xsd:date ;
  ke:endDate "2025-06-30"^^xsd:date ;
  ke:isEssential true ;
  ke:isRequested true .

ke:reward1 a ke:Reward ;
  ke:forContribution ke:contrib1 ;
  ke:currency ke:eCash ;
  ke:amount "150.00"^^xsd:decimal ;
  ke:date "2025-07-01"^^xsd:date .

ke:alice a ke:Agent ;
  ke:name "Alice" .
ke:bob a ke:Agent ;
  ke:name "Bob" .
ke:proj1 a ke:Project ;
  ke:name "Open Research" .
ke:eCash a ke:Cryptocurrency ;
  ke:name "eCash" .

Integration

Compatible with FOAF (agents), Dublin Core (metadata), and blockchain ontologies (transaction hashes).

Conformance

Systems must use RDF/OWL, adhering to defined classes and properties, with extensibility encouraged.
